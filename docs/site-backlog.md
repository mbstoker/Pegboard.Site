# Pegboard.Site — backlog

Open bug / polish / content items for the **marketing website** at `epegboard.com` (the public-facing site, not the app). The site's job is: get cold visitors to either the instant demo or sign-up; explain what ePegboard is; answer the "is this for me?" question for prospects who arrived via cold email / Reddit / search.

This is the site-scoped equivalent of `../../PegboardWeb/Docs/product/backlog.md` (product app) and `../../Pegboard.CRM/docs/crm-backlog.md` (internal CRM tools). Per the single-backlog principle, marketing-site work is its own tracker because it has its own audience (cold visitors), its own lifecycle (content/funnel-driven, not feature-driven), and its own deploy path.

## Conventions

- **id**: `SITE-NNN` sequential, don't recycle.
- **category**: `Bug` / `Feature` / `Polish` / `Content`.
- **priority**: `Now` / `Next` / `Later`. `Now` = visible to live prospects today and harming conversion; `Next` = wants done this month; `Later` = nice-to-have.
- **status**: `Open` / `In progress` / `Done` / `Dropped`.
- **source**: where the item came from (date + brief origin).
- **notes**: one-line summary of *what* + any locked-in decisions; point at the source for the full thread.
